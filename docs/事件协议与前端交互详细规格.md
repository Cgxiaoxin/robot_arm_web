版本：v1.0
适用：robot_arm_web（Socket.IO）
目标：让前端可稳定接入 move_p / move_l / jog，并可视化执行状态与错误。

1. 总体原则
前端只发“语义化运动命令”，不直接处理 IK/插值细节。
所有运动命令必须带：command_id、arm_id、motion_type、frame、target、constraints。
单位统一：
UI 输入：位置 mm、姿态 deg
发送到后端：位置 m、姿态 rad
前端必须实现“命令生命周期 UI”：
accepted -> planning -> running -> completed/failed/cancelled
执行中禁止重复触发同一控制面板，防止并发抢占导致“乱跳”。
2. 事件清单（Socket.IO）
2.1 前端 -> 后端
motion:execute：发送运动命令
motion:cancel：取消指定命令（或当前命令）
motion:pause：暂停
motion:resume：恢复
motion:query：查询当前执行状态
motion:set_default_constraints（可选）：设置默认约束参数
2.2 后端 -> 前端
motion:accepted：命令已受理
motion:rejected：命令被拒绝（参数/状态不合法）
motion:status：状态更新（planning/running/progress）
motion:completed：命令执行完成
motion:failed：命令失败（带错误码）
motion:cancelled：命令取消完成
motion:paused / motion:resumed
motion:telemetry（可选高频）：位姿、关节、误差、速度

5. 错误码与前端文案映射
ARM_NOT_CONNECTED -> 机械臂未连接，请先连接并初始化
IK_UNREACHABLE -> 目标位姿不可达，请减小位移或调整姿态
IK_LOW_ACCURACY -> IK 精度不足，请降低速度并接近可操作区
SINGULARITY_NEAR -> 接近奇异位形，请调整姿态后再试
JOINT_LIMIT_REJECTED -> 关节超限，命令已拒绝
CONSTRAINT_VIOLATION -> 速度/加速度参数超范围
COMMAND_PREEMPTED -> 命令被新命令抢占
CAN_TIMEOUT -> 通讯超时，请检查总线与供电
INTERNAL_ERROR -> 内部错误，请查看系统日志
前端展示规范：
[错误码] 中文文案（可折叠 detail）

6. 前端页面模块改造（可直接分配开发任务）
对应文件：frontend/index.html, frontend/js/main.js, frontend/css/style.css

6.1 “笛卡尔控制”Tab 结构
A. 模式栏
单选：PTP(move_p) / LINEAR(move_l) / JOG
当前执行状态灯：IDLE/RUNNING/PAUSED/FAULT
B. 目标位姿输入栏
X/Y/Z (mm) + Rx/Ry/Rz (deg)
frame 下拉（默认 BASE）
C. 约束参数栏
线速度、线加速度、角速度、角加速度
“恢复默认”按钮
D. 执行控制栏
执行、暂停、继续、取消、急停
E. Jog 区
XYZ/RPY 六轴点动按钮（支持长按连续）
步长选择 1/2/5/10/20 mm，姿态步长 0.5/1/2/5 deg
F. 反馈区
进度条、当前 pose、目标 pose、误差、最近错误
6.2 交互状态机（前端）
idle: 全部可编辑，执行按钮可用
accepted/planning: 禁止重复执行，显示“规划中”
running: 禁止改目标，允许 pause/cancel
paused: 允许 resume/cancel
completed/failed/cancelled: 回到 idle
7. 前端事件处理详细逻辑（JS 可照着写）
7.1 发命令前校验
arm 是否已连接
数值是否合法（非 NaN、范围合理）
LINEAR 模式下必须填完整 pose
位置输入 mm 转 m，姿态 deg 转 rad
7.2 发命令流程
生成 command_id（uuid）
组装 payload
socket.emit('motion:execute', payload)
UI 进入 pending（按钮禁用+显示 spinner）
7.3 收包流程
motion:accepted：记录 activeCommandId
motion:status：更新进度条和实时误差
motion:completed：toast success + 恢复 UI
motion:failed：toast error（含错误码）+ 恢复 UI
motion:cancelled：toast info + 恢复 UI
只处理 command_id == activeCommandId 的状态，避免串包。

8. 与现有事件兼容策略（过渡期必须）
前端保留旧接口，但内部统一改成新接口调用：

cartesian_move_to() -> 封装成 motion:execute + CARTESIAN_PTP
smooth_cartesian_move() -> 封装成 motion:execute + CARTESIAN_LINEAR
cartesian_jog() -> 封装成 motion:execute + CARTESIAN_JOG
后端在过渡期继续接收旧事件，但返回中提示 deprecated: true（前端可不显示，仅日志记录）。

9. UI 文案与标识规范（避免误解）
坐标显示旁固定注明：Frame: BASE（或当前 frame）
单位标签不可省略：mm / deg
模式清晰标注：
PTP：末端不保证直线
LINEAR：末端直线插补
臂标识建议：右臂(51~57)、左臂(61~67)，避免仅写 left/right
10. 联调验收用例（前端自测）
用例1：PTP 小位移（10mm）成功，状态完整闭环
用例2：LINEAR 小位移成功，进度持续更新
用例3：不可达点，返回 IK_UNREACHABLE
用例4：执行中点 cancel，收到 cancelled
用例5：连续快速下发命令，旧命令应 COMMAND_PREEMPTED
用例6：断开设备时发命令，返回 ARM_NOT_CONNECTED
