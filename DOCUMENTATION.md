## 1. 系统概述

本系统为双 7-DOF 机械臂提供统一的 Web 控制界面。操作人员只需通过浏览器即可完成连接、使能、轨迹播放、示教编程等全部工作流程，无需安装任何客户端软件。

**硬件架构：**

| 组件 | 说明 |
|------|------|
| 控制主机 | Linux PC（运行本系统） |
| CAN 接口 | 两路 CAN 总线（can0, can1），波特率 1 Mbps |
| 左臂（can1） | 7 轴电机，ID 61–67 |
| 右臂（can0） | 7 轴电机，ID 51–57 |
| 通讯协议 | CAN 2.0 / SocketCAN |

> **注意：** 界面上的"左臂"和"右臂"标签已校正为与机器人物理标注一致。程序内部的 `left`（can0）对应物理**右臂**，`right`（can1）对应物理**左臂**。

## 2. 功能特性

### 核心功能

| 功能 | 描述 |
|------|------|
| 🎬 **轨迹播放** | 选择已保存的轨迹文件，一键播放 / 暂停 / 停止，支持**循环播放** |
| 🎯 **预设动作** | 6 个大按钮快速触发预设动作（待机、打鼓、弹琴、挥手、鞠躬、自定义） |
| 📊 **实时监控** | 实时显示左右臂各 7 个关节的位置（弧度 / 角度）、限位范围和状态 |
| 🎮 **群控管理** | 支持仅左臂 / 仅右臂 / 双臂同步三种控制目标 |
| 🕹️ **手动控制** | 单关节点动控制（+ / − 按钮与滑杆），可选步长 0.01–1.0 rad |
| 📐 **笛卡尔控制** | 通过 XYZ 坐标控制末端执行器位置（增量 / 绝对坐标） |
| 📝 **示教编程** | 拖动机械臂记录点位 → 组合成轨迹 → 保存与回放 |
| ⚡ **速度设置** | 4 档预设速度 + 自定义速度 / 加速度 / 减速度 |
| 🛡️ **安全保护** | 关节限位检查、操作审计日志、紧急停止 |
| 🏠 **零点标定** | 标定当前位置为零点，一键回零 |

## 4. 安装部署

### 4.1 安装 Python 依赖

```bash
cd /data/coding_pro/robot_arm_web
pip install -r requirements.txt
```

### 4.2 配置 CAN 接口

**方式一：使用配置脚本（推荐）**

```bash
# 首次配置
sudo ./setup_can.sh

# 如遇 CAN 接口异常，使用重置脚本
sudo ./reset_can.sh
```

**方式二：手动配置**

```bash
# 配置右臂 CAN 接口 (can0)
sudo ip link set can0 down
sudo ip link set can0 type can bitrate 1000000
sudo ip link set can0 up

# 配置左臂 CAN 接口 (can1)
sudo ip link set can1 down
sudo ip link set can1 type can bitrate 1000000
sudo ip link set can1 up
```

### 4.3 验证 CAN 状态

```bash
ip link show can0 | grep state
ip link show can1 | grep state
# 应看到 "state UP"
```

## 6. 界面操作指南

### 6.1 页面总览

页面从上到下分为以下区域：

| 区域 | 位置 | 说明 |
|------|------|------|
| **顶部导航栏** | 页面顶部 | 连接 / 断开、启动使能、去使能、关闭电机、⚠ 紧急停止 |
| **轨迹播放** | 第一区块 | 轨迹文件选择 + 循环开关 + 播放 / 暂停 / 停止 / 回零 + 进度条 |
| **预设动作** | 第二区块 | A–F 六个快捷动作按钮 |
| **实时监控** | 第三区块 | 左右臂各 7 个关节的实时 状态表格，含位置（rad）、角度（°）、限位范围、状态 |
| **群控管理** | 第四区块 | 选择控制目标：仅右臂 / 仅左臂 / 双臂同步 |
| **Tab 功能区** | 第五区块 | 5 个功能选项卡（见下文） |

### 6.2 基本操作流程

```
连接 → 启动使能 → 选择控制目标 → 执行动作 → （退出前）回零 → 去使能 → 断开
```

**详细步骤：**

1. **连接机械臂** — 点击页面顶部的 `连接` 按钮，等待状态指示灯变为绿色 ● 已连接
2. **启动使能** — 点击 `启动使能`，等待电机初始化完成
3. **选择控制目标** — 在"群控管理"区域选择：
   - `仅右臂`（can0，电机 51–57）
   - `仅左臂`（can1，电机 61–67）
   - `双臂同步`（默认，同时控制两只臂）
4. **执行动作** — 通过以下任一方式：
   - 点击预设动作按钮（A–F）
   - 在轨迹播放区选择文件并播放
   - 在手动控制 / 笛卡尔控制中操作
5. **退出时** — 点击 `回零` → `去使能` → `断开`

### 6.3 轨迹播放

| 控件 | 功能 |
|------|------|
| 轨迹选择下拉框 | 选择 `trajectories/` 目录下的轨迹文件 |
| 🔁 循环开关 | 打开后轨迹将循环播放，直到手动停止 |
| ▶ 播放 | 开始播放（如已暂停，则恢复播放） |
| ⏸ 暂停 | 暂停播放，可继续 |
| ⏹ 停止 | 停止播放并复位 |
| 🏠 回零 | 所有关节回到零点位置 |
| 进度条 | 实时显示播放进度 |

**运行模式选择：** 页面右上角的下拉框可切换「位置模式」和「轨迹位置模式」。

### 6.4 预设动作

| 按钮 | 动作名称 | 对应文件 |
|------|---------|---------|
| A | 待机 | `preset_a.json` |
| B | 打鼓 | `preset_b.json` |
| C | 弹琴 | `preset_c.json` |
| D | 挥手 | `preset_d.json` |
| E | 鞠躬 | `preset_e.json` |
| F | 自定义 | `preset_f.json` |

> 预设文件存放在 `trajectories/` 目录，可手动编辑 JSON 文件修改动作内容。

### 6.5 手动控制（Tab 1）

功能：对左右臂的每个关节进行精确控制。

| 控件 | 说明 |
|------|------|
| **步长选择** | 0.01 / 0.05 / 0.1 / 0.5 / 1.0 rad |
| **−/+按钮** | 按步长递减 / 递增关节位置 |
| **0 按钮** | 将该关节回零 |
| **滑杆** | 拖动滑杆精确控制关节位置（范围 ±3.14 rad） |
| **自由拖动** | 开启后可直接手动拖动机械臂（适合示教录点位） |

### 6.6 笛卡尔控制（Tab 2）

功能：通过 XYZ 坐标控制末端执行器位置。

**增量控制：**
- 选择增量步长（5 / 10 / 20 / 50 / 100 mm）
- 选择控制臂（右臂 / 左臂）
- 使用 X / Y / Z 方向的 −/+ 按钮或滑杆移动末端

**绝对坐标控制：**
- 输入目标 X、Y、Z 坐标（mm）
- 点击「移动到目标」

**实时显示：**
- 两臂末端的当前 X、Y、Z 坐标（mm）实时更新

> 坐标系以关节零位时的末端位置为原点。

### 6.7 示教编程（Tab 3）

#### 零点标定

1. 使用"自由拖动"模式手动移动机械臂到物理零点位置
2. 点击 `🎯 标定当前位置为零点` 保存零点
3. 之后所有位置都相对于这个零点
4. 点击 `📥 查看零点偏移` 查看当前零点偏移值

#### 轨迹录制流程

```
新建轨迹 → 拖动机械臂到目标位置 → 记录当前位置 → 重复 → 设置参数 → 保存轨迹
```

**详细步骤：**

1. 点击 `📝 新建轨迹`，输入轨迹名称
2. 在轨迹编辑器中可设置：
   - **名称**：轨迹显示名
   - **描述**：可选备注信息
   - **循环播放**：勾选则播放时自动循环
   - **速度倍率**：0.1–5.0 倍速
3. 使用"自由拖动"或"手动控制"将机械臂移动到目标姿态
4. 输入点位名称（可选）和延迟时间（秒），点击 `📍 记录当前位置`
5. 重复步骤 3–4 录制多个点位
6. 点击 `▶ 预览播放` 查看效果
7. 点击 `💾 保存轨迹` 保存到 `trajectories/` 目录

#### 轨迹管理

| 按钮 | 功能 |
|------|------|
| 📝 新建轨迹 | 创建空轨迹并进入编辑模式 |
| 📂 加载轨迹 | 从已有文件加载进编辑器 |
| 🗑️ 删除轨迹 | 删除选中的轨迹文件 |
| 🔄 刷新列表 | 重新加载轨迹文件列表 |

### 6.8 速度设置（Tab 4）

| 预设 | 速度 (rad/s) |
|------|-------------|
| 极慢 | 0.2 |
| 慢速 | 0.5 |
| 中速 | 1.0（默认高亮） |
| 快速 | 2.0 |

**自定义速度：**
- 速度滑杆：0 – 2.0 rad/s 连续调节
- 精确输入：分别设置速度、加速度、减速度，点击「应用」

> ⚠ **首次使用务必设置为"极慢"或"慢速"**，确认动作正确后再提高速度。

### 6.9 系统日志（Tab 5）

实时显示系统运行日志，包含连接状态变更、操作记录、错误信息等。
- 点击 `清空` 按钮可清除日志显示。

### 6.10 紧急停止

> ⚠ **紧急停止按钮位于页面右上角，红色醒目标识。**

点击后所有电机将立即断电停止。在任何异常情况下，请立即点击此按钮。

## 7. 轨迹文件格式

轨迹文件为 JSON 格式，存储在 `trajectories/` 目录下。

### 完整示例

```json
{
  "name": "摆臂练习",
  "description": "左右臂交替摆动动作",
  "points": [
    {
      "name": "起始位",
      "positions": {
        "51": 0.8994,
        "52": 2.6779,
        "53": 0.8121,
        "54": 0.9551,
        "55": -0.3637,
        "56": 2.0844,
        "57": 1.1927,
        "61": -1.5892,
        "62": -1.1304,
        "63": -0.4911,
        "64": -2.9320,
        "65": -0.4208,
        "66": -1.4416,
        "67": -1.6922
      },
      "delay": 1.0
    },
    {
      "name": "终点位",
      "positions": {
        "51": 0.2400,
        "52": 2.6779,
        "...": "..."
      },
      "delay": 1.5
    }
  ],
  "loop": false,
  "speed_multiplier": 1.0,
  "created_at": "2026-03-04T10:46:08"
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 轨迹显示名称 |
| `description` | string | ❌ | 轨迹描述信息 |
| `points` | array | ✅ | 轨迹点数组 |
| `points[].name` | string | ❌ | 点位名称 |
| `points[].positions` | object | ✅ | 电机位置表 `{ "电机ID": 弧度值, ... }` |
| `points[].delay` | number | ❌ | 到达后等待时间（秒，默认 1.0） |
| `loop` | boolean | ❌ | 是否循环播放（默认 `false`，可被前端开关覆盖） |
| `speed_multiplier` | number | ❌ | 速度倍率（默认 1.0） |
| `created_at` | string | ❌ | 创建时间（ISO 8601 格式） |

### 电机 ID 对照表

| 臂 | CAN 通道 | 电机 ID | 关节 |
|----|---------|---------|------|
| 右臂 | can0 | 51–57 | 肩关节 → 腕关节 |
| 左臂 | can1 | 61–67 | 肩关节 → 腕关节 |

## 8. 配置参考

配置文件位于 `backend/config.py`，可根据需要修改以下参数：

### CAN 配置

```python
DEFAULT_LEFT_CAN = "can0"      # 右臂（物理）CAN 通道
DEFAULT_RIGHT_CAN = "can1"     # 左臂（物理）CAN 通道
CAN_BITRATE = 1000000          # CAN 波特率 (1 Mbps)
```

### 电机配置

```python
LEFT_MOTOR_IDS = [51, 52, 53, 54, 55, 56, 57]   # 右臂电机 ID
RIGHT_MOTOR_IDS = [61, 62, 63, 64, 65, 66, 67]   # 左臂电机 ID
```

```python
HOST = "0.0.0.0"               # 监听地址（0.0.0.0 表示所有网卡）

STATE_UPDATE_INTERVAL = 200    # 状态推送间隔（毫秒）
```

```python
SPEED_PRESETS = {
```

```
│  ├────────────┬─────────────┤   │
│  │ CartesianCtrl │ SafetyCtrl │   │
│  │ (FK/IK运算)  │(限位/审计)  │   │
│  └────────────┴─────────────┘   │
└────────────┬────────────────────┘
             │
             ▼  CAN 总线 (SocketCAN)
┌────────────┬────────────────────┐
│  can0      │       can1         │
│  右臂      │       左臂         │
│  电机51-57 │       电机61-67     │
│  (7-DOF)   │       (7-DOF)      │
└────────────┴────────────────────┘
```

### 通讯协议

| 层级 | 技术 | 说明 |
|------|------|------|
| 浏览器 ↔ 后端 | Socket.IO（WebSocket） | 双向实时通讯，200ms 间隔推送状态 |
| 后端 ↔ 电机 | CAN 2.0（SocketCAN） | 1 Mbps，Linux 原生驱动 |
| 并发模型 | eventlet（协程） | 非阻塞 IO，避免CAN读取阻塞事件循环 |

### 后端核心模块

| 模块 | 职责 |
|------|------|
| `app.py` | Flask 路由、Socket.IO 事件处理器、状态广播循环 |
| `group_controller.py` | 双臂协调控制、轨迹播放工作线程、关节读写 |
| `cartesian_controller.py` | 7 自由度正/逆运动学（纯 Python 实现，Rodrigues 公式 + 阻尼最小二乘法） |
| `safety_controller.py` | 关节限位检查、速度监控、安全事件日志 |
| `trajectory_engine.py` | 轨迹文件的 CRUD 操作、轨迹合并、时长计算 |
| `arm_control.py` | 底层 CAN 通讯封装（单臂级别） |

```bash

```

1. 终止所有 CAN 相关进程
2. 关闭 CAN 接口
3. 加大发送队列（txqueuelen=100）重新配置并启动
4. 验证状态

## 12. 常见问题 FAQ

### 连接与通讯

| 问题 | 原因及解决方法 |
|------|---------------|
| 无法连接 | ① 检查 CAN 线是否插好 ② 检查电源 ③ 运行 `ip link show can0` 确认 CAN 接口状态为 UP |
| 频繁断开重连 | ① 确认只运行了一个服务实例 ② 运行 `./reset_can.sh` 重置 CAN 接口 |
| 电机无响应 | ① 检查电机是否上电 ② 运行 `python3 tools/diagnose_can.py` 诊断 |

### 操作问题

| 问题 | 解决方法 |
|------|---------|
| 动作不同步 | ① 重启程序 ② 确认选择了"双臂同步"控制目标 |
| 浏览器打不开 | 手动打开浏览器访问 `http://127.0.0.1:5000` |
| 修改代码后无效 | ① 后端：需要 `Ctrl+C` 停止服务并重新运行 `./start.sh` ② 前端：`Ctrl+F5` 强刷浏览器 |
| 轨迹播放卡住 | ① 确保不是 CAN 总线拥堵 ② 尝试停止播放后重新开始 |
| 循环播放不生效 | 确保打开循环开关后重新 **重启后端服务** 并刷新浏览器 |

### 性能问题

| 问题 | 解决方法 |
|------|---------|
| 页面卡顿 | 减小 `STATE_UPDATE_INTERVAL`（状态推送间隔）或关闭不需要的浏览器标签 |
| CAN 总线超时 | 运行 `./reset_can.sh` 重置，确保无其他程序占用 CAN 接口 |

## 13. 安全注意事项

> [!CAUTION]
> 在操作机械臂前，务必仔细阅读以下安全须知。

1. **首次使用务必设置慢速** — 在"速度设置"中选择"极慢"（0.2 rad/s）
2. **确保周围无障碍物** — 机械臂运行范围内不得有人员或物品
3. **熟记紧急停止位置** — 右上角红色 `⚠ 紧急停止` 按钮
4. **退出前务必回零** — 点击 `🏠 回零` 将机械臂归位
5. **修改配置需谨慎** — 关节限位（`JOINT_LIMITS`）的修改可能导致机械碰撞
6. **定期检查零点** — 长期使用后零点可能偏移，需重新标定
7. **保持 CAN 线缆连接牢固** — 松动可能导致通讯中断、动作失控
